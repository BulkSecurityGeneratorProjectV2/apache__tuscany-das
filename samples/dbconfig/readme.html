Tuscany RDB DAS Database Setup Utility

 

This utility provides a jar file to be used by RDB DAS sample applications (web or standalone) to do database setup. 


So far  the samples provided canned pre-created Derby database with tables and data.   There was no convenient way to  refresh  table  data  during  the samples  execution.  This utility  supports Derby database at present.

 
The  Database Setup Utility uses a config file to get the setup information and exposes following APIs:-

public DBInitHelper(String ConfigFileLocation)

public DBInitHelper(InputStream ConfigFileStream)

public void initializeDB(boolean force) - Create database tables and fill data. If force=true, all tables will be dropped and recreated.

public void createSchema(boolean force) - create database tables (with force=true, if a table pre-exists, it will be dropped and recreated)

public void boolean deleteSchema() - delete all database tables, returns false on failure

public void refreshData(boolean force) - populate tables data (with force=true, if data pre-exists, it will be deleted and fresh data will be inserted)

public boolean deleteData() - delete data from tables, return false on failure

public boolean isSchemaCreated() - will return true, if the tables exist in the database

public boolean isDataCreated() - will return true, if the tables have data


Assumptions:

TableList elemenr lists tables in proper sequence, i.e. parent tables first and then child tables. This sequence will be used in create,insert,delete,drop
to take care of referential integrity.

The config file required by utility supports the following features through different attributes and elements. Sample xml file will be shown later:-

createOrImport

This attribute can have one of the two values- create or import. If it is set to create, the hardcoded classes inside the utility jar

like CompanyData, DepartmentData will be used. If it is set to import, user needs to supply <TablesData> element in the config and data in tables

will be populated based on the values specified in this element.

ConnectionInfo

This element provides connection specific information - like vendor specific database URL, user name, password etc. If the DataSource is

provided by the web container, user needs to only fill dataSource attribute in this. In case of standalone J2SE samples, user needs to fill ConnectionProperties
element inside ConnectionInfo. These two ways of connection specification are mutually exclusive.

TableList

  This element needs to specify the table names required by the sample. Only these tables will be considered for creation, data population. This

gives flexibility to the user to chose the required set of tables based on the sample's requirement.

TablesData

This element is optional. It will be used only in case of createOrImport=import. In this, the table name, column names, data types and data

can be conveniently specified by the user (check the sample xml given below). Advantage of using import instead of create is flexibility over the set of data

to be used for the tables.


Set Up


To use this utility, include its jar and required database driver's jar in the classpath. Provide the config file used by this utility , similar to the example below, in the source folder of the sample.
